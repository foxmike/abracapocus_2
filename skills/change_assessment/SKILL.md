# Change Assessment Skill

- **Purpose**: Compare new outputs against plan intent and prior state to determine whether goals were met.
- **Inputs**: plan diffs, task registry, backend stdout/stderr, review findings.
- **Outputs**: assessments that inform state updates, risk tracking, and operator approvals.
- **Usage**: management and reviewer agents reference this skill when updating runtime state or producing review summaries.
- **Future Work**: integrate deterministic diffing and semantic validation of generated artifacts.
