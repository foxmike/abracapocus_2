# Task Handoff Skill

- **Purpose**: Capture execution summaries, remaining work, and instructions for the next agent or operator.
- **Inputs**: task metadata, backend result, review/verifier status.
- **Outputs**: structured handoff blocks stored with reports and runtime state.
- **Usage**: Management + verifier agents rely on this skill to keep historical continuity when multiple runs touch the same task.
- **Future Work**: file attachments, operator acknowledgment workflow, and integration with external ticketing systems.
